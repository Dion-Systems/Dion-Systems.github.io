What is Dion?

Dion is our project to build a better format for storing structured information on computers, which notably includes (but is not limited to) the code that we use to program computers. We've identified problems with using text files as the base format for this purpose, so our goal is to provide an alternative format to text files, and to also provide tools to work with that format. We think that this can help make our programming tools smarter, more accessible, easier to learn how to use, faster, and less of a pain in general.

Is Dion a language? Is it an editor? Is it a compiler? Really, what is Dion?!

Dion is more-or-less a name we've given to the entire project of shifting the fundamental axioms of our programming systems in favor of something that—we hypothesize—can be much better. This whole project may include many tools. Our original demo at Handmade Seattle presented a prototype that featured a tightly coupled system, where all three of these things lived very close to each other. We then learned how to get many of the benefits we presented in the original Handmade Seattle demo, but without tightly coupling all of these systems, and it involves a new format for code. So, these days, "Dion" refers to the project of building that new format, and the tools around it.

What is the business model of Dion?

The Dion team is currently comprised of three part-time volunteers who work on the project in our free-time, so there isn't really a business model currently. We do not make any money from Dion right now, and we will continue to work on it as volunteers for the time being. Our future plans for a business model are to-be-determined, but we know that much of our work will be public and open.

Will Dion be open-source?

We expect to publish many things as open or open-source, most notably the Dion format specification itself. We also plan to release our metaprogramming utility Metadesk under the MIT license. We have not decided the answer to this question for all code we write as part of the Dion project; it is certainly possible that certain projects we work on will not be open-source.

How will version control work with a new format?

Integrating with existing programming workflows is crucial when developing programming tools. However, the goals of Dion make this challenging. That being said, we hope to integrate with existing version control systems to the extent that we can, using custom diff and merge utilities for the new format. The diff and merge implementation will obviously not take the same form as it does with text files (one possible avenue is a graph-based diff), but this does not prevent us from, for example, working with Git. More details will become available as we get further in our research and development.

What is Metadesk?

Metadesk is an open-source tool that we developed for metaprogramming, data generation, and code generation. The name "Metadesk" really applies to two things: A data description language, and a library that we developed for working with that language. That library includes utilities for parsing strings of the Metadesk language to an abstract syntax tree, introspecting on those abstract syntax trees, doing code generation, and more.

How does Metadesk relate to the Dion Format?

These projects are only related in the sense that Metadesk is helping us build the Dion Format, and both projects are being built with a very similar philosophy behind them. Metadesk is not the Dion Format, it is purely a utility that we're using for the current programming systems that we are all used to.
